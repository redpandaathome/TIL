인터넷을 사용할때 약속 된 기기간 통신하는 방식

TCP; 전송제어프로토콜
(Transmission Control Protocl)
서버 - 클라이언트간 데이터 전달 위한 프로토콜
잘게 나눈 데이터(패킷) 형태로 전송.
패킷은 여러 경로로 전달 (순서가 바뀌거나 손상될 가능성 존재)
header에 순서정보/ 손상여부(checksum) -> 재전송 요청

IP; 인터넷 프로토콜
인터넷 상의 주소(출발/도착)
(Ipv4(32bit)->Ipv6(128bit))
172.16.254.1
(172 -> 10101100)
4 * 8bit = 32 bits(4 bytes)
